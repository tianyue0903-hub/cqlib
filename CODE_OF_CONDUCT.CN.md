# Cqlib 行为准则

[English](CODE_OF_CONDUCT.md)

Cqlib 致力于为所有参与者提供开放、专业、友好和尊重的协作环境。无论是提交 Issue、Pull Request、参与讨论、撰写文档，还是在其他项目空间中代表 Cqlib 交流，所有参与者都应遵守本行为准则。

## 我们的承诺

项目维护者和社区成员承诺共同维护一个不受骚扰、歧视和恶意攻击影响的协作环境。我们欢迎不同背景、经验水平和观点的开发者参与 Cqlib 项目。

## 适用范围

本行为准则适用于所有 Cqlib 项目空间，包括但不限于：

- Issue、Pull Request 和代码评审
- Gitee、GitHub 或其他代码托管平台上的项目讨论
- 项目文档、示例和社区材料
- 邮件、会议、即时通信群组等项目相关沟通
- 代表 Cqlib 项目进行的公开或私下交流

## 鼓励的行为

我们鼓励以下行为：

- 使用友善、专业和包容的语言
- 尊重不同观点、经验和技术背景
- 聚焦事实、代码、设计和用户影响进行讨论
- 主动给出清晰、可执行的反馈
- 接受建设性批评，并在必要时调整方案
- 对新贡献者保持耐心，帮助他们理解项目流程
- 在发现问题时优先推动解决，而不是指责个人

## 不可接受的行为

以下行为不可接受：

- 人身攻击、侮辱、嘲讽、威胁或恐吓
- 基于身份、地域、性别、年龄、残障、宗教、民族、国籍、性取向或其他个人特征的歧视性言论或行为
- 性骚扰、不受欢迎的性化语言或图像
- 未经允许公开他人的私人信息，例如真实姓名、地址、邮箱、电话号码或工作单位信息
- 恶意刷屏、持续打断讨论、挑衅或扰乱项目协作
- 明知不实仍传播误导性信息
- 以贡献、评审或维护权限为条件施压他人
- 其他维护者认为不利于项目社区健康发展的行为

## 举报方式

如果你遇到或发现违反本行为准则的行为，请通过邮件联系项目维护者：

```text
tianyan@chinatelecom.cn
```

举报时请尽量提供：

- 事件发生的时间和位置
- 相关链接、截图或上下文
- 涉及的账号或人员
- 你希望维护者采取的处理方式

维护者会尽量保护举报者和相关人员的隐私，并根据实际情况进行调查和处理。

安全漏洞请不要通过行为准则流程公开讨论，请参考 [SECURITY.CN.md](SECURITY.CN.md)。

## 执行措施

维护者有权根据违规行为的严重程度采取适当措施，包括但不限于：

- 私下提醒或正式警告
- 要求修改或删除不当内容
- 隐藏、编辑或删除评论、Issue、PR 或其他项目内容
- 暂时限制参与项目讨论或贡献
- 永久禁止参与项目社区
- 在必要时向相关平台或组织报告严重违规行为

执行措施会尽量保持公平、一致，并以维护项目协作环境为目标。

## 维护者责任

项目维护者负责解释和执行本行为准则。维护者应避免滥用权限，并在处理争议时尽量基于事实、上下文和项目利益做出判断。

维护者自身也必须遵守本行为准则。如果维护者违反准则，其他维护者或项目负责人应按同等标准处理。

## 版本说明

本行为准则参考了开源社区常见行为准则的结构，并根据 Cqlib 项目的协作方式进行了调整。
