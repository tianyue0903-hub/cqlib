# Cqlib 安全策略

[English](SECURITY.md)

Cqlib 重视项目和用户的安全。如果你发现安全漏洞、供应链风险或可能影响用户运行环境的问题，请不要公开提交 Issue、Pull Request 或讨论帖。

请发送邮件到 <tianyan@chinatelecom.cn> 私下报告。项目官网是 <https://qc.zdxlz.com/>。

## 报告范围

请通过安全邮箱报告以下类型的问题：

- 可能导致任意代码执行、越权访问或数据泄露的问题
- 依赖、构建、发布或安装流程中的供应链风险
- Python 绑定、C 绑定或 FFI 边界中的内存安全问题
- 解析外部输入、加载文件、处理量子线路或 IR 时可能触发的拒绝服务问题
- 可能影响发布包完整性、构建产物可信度或用户环境安全的问题

普通 bug、功能建议、文档问题和非安全性的兼容性问题，请通过正常 Issue 或 Pull Request 流程反馈。

## 报告内容

为了便于确认和修复，请尽量提供：

- 受影响的版本、commit 或发布包
- 受影响的平台、操作系统、CPU 架构、Rust 版本和 Python 版本
- 漏洞类型和影响范围
- 最小复现步骤或可运行示例
- 相关日志、崩溃信息、堆栈或输入样本
- 你认为可能的缓解方式或修复建议
- 是否已向其他平台、组织或数据库报告

请不要在报告中包含无关的个人数据、生产环境密钥、真实用户数据或无法授权分享的第三方材料。

## 处理流程

维护者收到报告后会尽快确认，并根据问题影响范围进行评估、修复和发布。处理过程中可能会联系报告者补充复现信息、确认修复效果或协调披露时间。

在修复发布前，请不要公开漏洞细节、利用代码或可复现攻击步骤，以免影响用户安全。

## 支持版本

项目仍处于早期版本阶段。一般情况下，安全修复会优先进入当前主分支和最新发布版本。对于旧版本是否回补修复，将根据漏洞严重程度、修复复杂度和实际维护成本决定。

## 致谢

如果报告者希望公开署名，维护者可以在修复发布后通过发布说明、致谢列表或其他合适方式表示感谢。是否公开署名以报告者意愿和实际披露安排为准。
