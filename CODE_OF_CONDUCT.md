# Cqlib Code Of Conduct

[简体中文](CODE_OF_CONDUCT.CN.md)

Cqlib is committed to providing an open, professional, friendly, and respectful collaboration environment for all participants. Whether you are opening issues, submitting pull requests, participating in discussions, writing documentation, or representing Cqlib in other project spaces, you are expected to follow this Code of Conduct.

## Our Pledge

Project maintainers and community members pledge to maintain a collaborative environment free from harassment, discrimination, and malicious personal attacks. We welcome developers with different backgrounds, experience levels, and viewpoints to participate in Cqlib.

## Scope

This Code of Conduct applies to all Cqlib project spaces, including but not limited to:

- Issues, pull requests, and code review
- Project discussions on Gitee, GitHub, or other code hosting platforms
- Project documentation, examples, and community materials
- Email, meetings, instant messaging groups, and other project-related communication
- Public or private communication while representing the Cqlib project

## Expected Behavior

We encourage the following behavior:

- Use friendly, professional, and inclusive language
- Respect different viewpoints, experience levels, and technical backgrounds
- Discuss facts, code, design, and user impact
- Provide clear and actionable feedback
- Accept constructive criticism and adjust proposals when appropriate
- Be patient with new contributors and help them understand project workflows
- Focus on solving problems instead of blaming individuals

## Unacceptable Behavior

The following behavior is unacceptable:

- Personal attacks, insults, mockery, threats, or intimidation
- Discriminatory language or behavior based on identity, region, gender, age, disability, religion, ethnicity, nationality, sexual orientation, or other personal characteristics
- Sexual harassment or unwelcome sexualized language or imagery
- Publishing another person's private information without permission, such as legal name, address, email address, phone number, or workplace information
- Spam, repeated disruption of discussions, provocation, or behavior that interferes with project collaboration
- Knowingly spreading misleading information
- Pressuring others by using contribution, review, or maintainer privileges
- Other behavior that maintainers consider harmful to the health of the project community

## Reporting

If you experience or observe behavior that violates this Code of Conduct, contact the project maintainers by email:

```text
tianyan@chinatelecom.cn
```

When reporting, please provide as much of the following as possible:

- Time and location of the incident
- Relevant links, screenshots, or context
- Accounts or people involved
- The action you would like maintainers to take

Maintainers will make reasonable efforts to protect the privacy of reporters and involved parties, and will investigate and respond based on the situation.

Do not use the Code of Conduct process to publicly discuss security vulnerabilities. See [SECURITY.md](SECURITY.md) instead.

## Enforcement

Maintainers may take appropriate action based on the severity of the violation, including but not limited to:

- Private reminder or formal warning
- Requesting that inappropriate content be edited or removed
- Hiding, editing, or deleting comments, issues, PRs, or other project content
- Temporarily restricting participation in project discussions or contributions
- Permanently banning participation in the project community
- Reporting serious violations to relevant platforms or organizations when necessary

Enforcement should be fair and consistent, with the goal of maintaining a healthy collaboration environment.

## Maintainer Responsibilities

Project maintainers are responsible for interpreting and enforcing this Code of Conduct. Maintainers should avoid abusing their privileges and should resolve disputes based on facts, context, and project interests.

Maintainers must also follow this Code of Conduct. If a maintainer violates it, other maintainers or project leads should handle the case under the same standards.

## Version Note

This Code of Conduct follows the structure commonly used in open source communities and has been adjusted for Cqlib's collaboration model.
